Solana DeFi Ecosystem Dashboard

This dashboard provides real-time monitoring of the ONyc token ecosystem across four Solana DeFi protocols. Every chart is fed by a live data pipeline that ingests on-chain transactions and account state changes, decodes them, and rolls them into time-series views refreshing every few seconds.

Use the View selector in the header to navigate between pages. Each page focuses on a different protocol or cross-protocol perspective. The Last dropdown controls the lookback window for every chart on the current page — from 1 hour to 90 days. Most charts have an expand button (↗) in the lower-right corner for a full-screen view.

ONyc Token Ecosystem

What is Monitored

ONyc is a Solana SPL token whose DeFi footprint spans decentralised exchanges, lending markets, and yield tokenisation protocols. This dashboard tracks every on-chain interaction across those venues in real time.

  • Token supply & derivative issuance (SY, PT, YT)
  • Liquidity depth & LP positions across DEX pools
  • Lending supply, borrow rates & collateral utilisation
  • Yield curves — base token yield, implied APY, lending APY
  • Protocol-level TVL & 24h activity volumes

Key Interactions Tracked

The pipeline captures four categories of on-chain events, decoded directly from Solana transaction instructions:

  • Swap events — buy / sell volumes on Orca & Raydium
  • Liquidity events — deposits, withdrawals, position changes
  • Lending events — supply, borrow, repay, liquidation
  • Yield events — PT/YT mints, redeems, and trades on Exponent

Cross-Protocol Analysis

Beyond per-protocol metrics, the dashboard integrates data across all four venues to surface ecosystem-level patterns and cross-protocol risks:

  • Aggregate supply — token distribution across DEX pools, lending reserves and yield vaults
  • Sell-impact estimates — combine liquidity depth with swap activity
  • Yield curve comparison — lending APY, implied PT rates and base token yield
  • Issuance flow — tracking from mint through DEX, lending and yield tokenisation

How It Works

Pipeline Overview

1

Ingest

Real-time Yellowstone gRPC streams plus periodic RPC snapshots capture every relevant on-chain event and account state change.

2

Decode & Enrich

Raw transactions are Borsh-decoded, matched against protocol discriminators, and enriched with pool/market metadata.

3

Store & Roll Up

Events land in TimescaleDB hypertables partitioned by time. Continuous aggregates refresh every 5 seconds for sub-minute freshness.

4

Serve

Parameterised view functions feed a lightweight API. The dashboard fetches each widget via HTMX with staggered loads and automatic refresh.

Data Notes

Historical Coverage

  • Real-time ingestion — continuous monitoring of on-chain events and account balances began on 16 March 2026. All data from this date forward is captured live via the streaming pipeline.
  • Backfilled history — balances prior to 16 March have been reconstructed from transaction data sourced via the Solscan API. Reconstruction has been extensively validated, though minor drift remains possible. Production installs requiring deep historical coverage would typically source from long-running indexers such as Dune or Allium.
  • Kamino obligation history — the backfill reconstruction does not extend to individual obligation balances on Kamino. Historical obligation-level data is available only from the live-ingestion start date onward.
